# FlockOS

> *"I am the vine; you are the branches."* — John 15:5

A complete church management platform built on Google Apps Script and Google Sheets. No server, no database subscription, no infrastructure — just a Google Sheet, a deployed Apps Script Web App, and a static front-end served via GitHub Pages.

---

## What It Is

FlockOS is a single-codebase, multi-church CRM and ministry platform designed to run inside any church's existing Google Workspace. One unified API endpoint backed by a 200-tab Google Sheet handles members, care cases, prayer, giving, attendance, events, songs, missions, and more. The front-end is a progressive web app hosted on GitHub Pages.

---

## Architecture

| Layer | Technology | Details |
|-------|-----------|---------|
| **Database** | Google Sheets | 200 tabs, one Sheet per church |
| **API** | Google Apps Script | One unified Web App endpoint |
| **Front-end** | HTML + Vanilla JS | GitHub Pages, no build step |
| **Auth** | SHA-256 + salt + pepper | RBAC with 6 permission levels |
| **Offline** | Service Worker + IndexedDB | Full offline capability via TheWellspring |

## Multi-Church Build System

Each church deployment is driven by a JSON config file. The build script copies the source tree and brands it per-church.

### Config schema (`FlockOS/Tools/Active Deployments/ChurchTemplate.json`)

```json
{
  "id": "",
  "name": "",
  "shortName": "",
  "brandName": "",
  "tagline": "Church Management & Ministry Platform",
  "favicon": "",
  "portrait": "",
  "themeColor": "#e8a838",
  "backgroundColor": "#1a1a2e",
  "databaseUrl": "",
  "photosUrl": "",
  "adminEmail": "",
  "analyticsId": "",
  "version": "3.0"
}
```

### Build command

```bash
bash FlockOS/Tools/Development\ Scripts/build_churches.sh
```

### What it does

1. Reads each `.json` config in `Active Deployments/` (skips `ChurchTemplate.json`)
2. Copies source tree + root files into `Church/<shortName>/`
3. Replaces database URL, tagline, theme/background colors via `sed`
4. Replaces logo/favicon (only when config specifies; blank = keep default)
5. Replaces `<title>` and `og:title` in all HTML files with church name
6. Replaces topbar brand text with `brandName`
7. Updates `manifest.json` — name, short_name, description, colors, icons

### Adding a new church

1. Copy `ChurchTemplate.json` → `flockos-yourchurch.json`
2. Fill in all fields
3. Optionally add a logo to `FlockOS/Images/`
4. Run `build_churches.sh`
5. Commit and push — church is live at `Church/<shortName>/`

## Deploying a New Church Backend

Full step-by-step instructions are in `FlockOS/Pages/the_pentecost.html`. Short version:

### 1. Create the Google Sheet

Create a Google Sheet named **FlockOS — [Church Name]**. Copy the Sheet ID from the URL.

### 2. Deploy the GAS backend

1. Create a new project at [script.google.com](https://script.google.com)
2. Open the project from within your Google Sheet (Extensions → Apps Script)
3. Paste `Master Code.gs` as `Code.gs`
4. Fill in the three church fields at the top of the file:
   ```js
   churchName:   'Grace Community Church',
   timezone:     'America/Chicago',
   ```
5. Run **`setupFlockOS`** from the function dropdown — builds all 200 tabs, seeds content, creates the admin account, and installs the care email triggers in one run
6. Deploy → **New Deployment → Web App** (Execute as: Me, Who has access: Anyone)
7. Copy the Web App URL, paste it into `DEPLOY_CONFIG.churchAppUrl`, then run **`registerChurchUrl`**

### 3. Connect the front-end

Add the Web App URL as `databaseUrl` in the church's JSON config, then run `build_churches.sh`.

## Authentication

| Property | Value |
|----------|-------|
| Method | Email + passcode |
| Hashing | SHA-256 · per-user salt · server-side pepper |
| Session TTL | 6 hours (configurable in AppConfig) |
| RBAC levels | `readonly` (0) · `volunteer` (1) · `care` (2) · `leader` (3) · `pastor` (4) · `admin` (5) |

## Key Features

- **Member Management** — Full CRUD with pastoral notes, tags, contact masking
- **Member Cards** — Sequential card numbers with configurable prefix
- **Pastoral Care** — 13 care types with per-type workflow guides, interaction logging, follow-up reminders, and daily pastoral summary emails
- **Prayer** — Public submission + admin management, status, assignment, replies
- **Compassion & Outreach** — Benevolence fund, needs tracking, campaigns
- **Discipleship** — Growth paths, milestones, courses, quizzes
- **Daily Devotional** — Scripture, reflection, reading plan, journal, prayer — one view
- **Learning Hub** — 16-tab education portal: courses, quizzes, theology, lexicon, apologetics, counseling, devotionals, reading plans, certificates
- **World Missions** — 48 country dossiers, partner tracking, team management
- **Music Stand** — Song CRUD, ChordPro charts, arrangement management, live setlist, PDF export
- **Calendar** — Month/week/day/agenda, personal events, iCal feeds, recurrence, task management
- **Service Planning** — Order builder with item scheduling
- **Attendance & Check-In** — Small groups, Bible studies, event check-in
- **Communications** — Email/SMS templates, campaign tracking, delivery logs
- **Volunteers** — Scheduling, team assignments, vCard download
- **Statistics** — Attendance, giving, growth analytics dashboards
- **Themes** — 13 built-in themes, personal theme picker, Interface Studio (30+ controls)
- **Audit Logging** — All interactions tracked with user, action, timestamp
- **Offline Mode (TheWellspring)** — Full offline capability via IndexedDB — import an `.xlsx` of the database, all 200 tabs served locally
- **Google Drive Sync (TheWell)** — Auto-sync church data backed by Drive `.xlsx` files
- **Automated Pastoral Emails** — Care follow-up reminders, escalation alerts, daily 6 AM pastoral summary
- **Module Permissions** — Per-user GRANT/DENY overrides via Permissions sheet
- **iOS Ready** — All inputs 16px, responsive modals, root font-size: 100% for user scaling
- **Multi-Church** — Unlimited church deployments from one codebase, each with independent branding and database
