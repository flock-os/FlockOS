🚀 Deployment Steps
1
Create the GAS Project

Go to script.google.com. Create a new project. Name it (e.g., TBC — FlockOS Backend).

2
Paste Code.gs

Open 10-Master Code.md. Copy everything. In Apps Script, replace all of Code.gs with the contents. Save.

3
Add FirestoreSync.gs

In Apps Script, click + to add a new file. Name it FirestoreSync.gs. Paste the copied block in. Save. This handles hourly Firestore → Sheet backups.

4
Add SyncHandler.gs

Same — create SyncHandler.gs and paste in the copied block. Save. This receives real-time sync writes from the Cloud Function.

5
Set Script Properties

Copy the generated block above. Paste it into a new file in Apps Script (e.g., Setup.gs). Save, then run setAllScriptProperties(). You'll be prompted to authorize.

6
Delete Setup.gs immediately

After the function runs successfully, delete Setup.gs before you do anything else. It contains plaintext credentials.

7
Run setupFlockOS()

Back in Code.gs, run the setupFlockOS() function. It seeds all defaults and creates the admin user. Check the Execution Log for success.

8
Delete ADMIN_PASSWORD

After setup completes, go to ⚙ Project Settings → Script Properties. Find ADMIN_PASSWORD and delete it.

9
Deploy as Web App

Deploy → New Deployment → Web App. Set "Execute as: Me" and "Who has access: Anyone". Copy the Web App URL.

10
Register the URL

Paste the Web App URL into CHURCH_APP_URL above, copy the regenerated block, run again (only the URL property will change). Then run registerChurchUrl() in GAS.

✓
Register in Command Center

Return to the Great Commission and click + Register to add this church to the network.
