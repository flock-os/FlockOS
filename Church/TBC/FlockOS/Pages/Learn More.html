What Is FlockOS?

FlockOS is an open-source Church Relationship Management (CRM) system designed for churches of any size. It runs entirely on Google's free infrastructure — Google Sheets as the database, Google Apps Script as the backend, and static HTML/JS as the frontend. There are no recurring hosting costs, no complex server setups, and no proprietary data formats.

FlockOS manages your entire church operations: membership directory, attendance tracking, pastoral care, communications, giving records, event management, worship planning, missions, small groups, and much more — all within a single unified Google Sheet with 200 tabs powered by one Google Apps Script backend.

How It Works

Architecture overview: FlockOS uses a single Google Sheet as its database (200 tabs), exposed through one unified Google Apps Script Web App endpoint. The frontend is a single-page application (SPA) built in vanilla JavaScript — no frameworks, no build tools, no dependencies.

Unified Database

One Google Sheet contains all 200 tabs, served by a single Apps Script backend (Master Code.gs). The tabs span four functional areas:

  • CRM & Operations — Members, auth, pastoral care, services, songs, communications, giving, events, groups, volunteers, and more.
  • Public Content — Devotionals, Bible encyclopedia, reading plans, quiz, theology. Read-only routes, no authentication required.
  • Missions — Persecution data, country dossiers, mission teams, field reports. Specialized global outreach module.
  • Statistics — Analytics, growth metrics, and expansion slots for future features.

Frontend Layer

Two main HTML applications serve different audiences:

  • The Good Shepherd (admin) — Full CRM with 48+ modules, role-gated navigation, notification system, and dashboard cards. For church staff and volunteers.
  • Public Site (index.html) — Member-facing app with devotionals, reading plans, prayer requests, learning hub, ministry hub, and community features. Auth-scoped navigation reveals modules as users log in.

Getting Started

  1. Create one Google Sheet — this single spreadsheet will hold all 200 tabs as your unified database.
  2. Create an Apps Script project — open Extensions → Apps Script from your Sheet.
  3. Copy Master Code.gs.md — paste the unified backend file (FlockOS/Tools/Master Deployment/Master Code.gs.md) as Code.gs in your Apps Script project.
  4. Run setupFlockOS() — the setup function creates all 200 tabs with correct headers automatically.
  5. Deploy as a Web App — publish the Apps Script project. Set access to “Anyone” to allow both public and authenticated routes through the single endpoint.
  6. Run registerChurchUrl() — paste your deployed Web App URL into DEPLOY_CONFIG.churchAppUrl, then run this function to register the URL in your database.
  7. Create your first admin account — run the initial user setup to create a pastor-level admin account.
  8. Invite your team — add staff and volunteers with appropriate role levels.

Folder Structure

Security

FlockOS implements enterprise-grade security on free infrastructure:

  • SHA-256 with salt & pepper — passwords are never stored in plain text
  • 6-tier RBAC — readonly, volunteer, care, leader, pastor, admin
  • Session tokens — time-limited authentication tokens with automatic expiry
  • Confidential case gating — sensitive pastoral care cases restricted to pastor+ roles
  • Audit logging — every data modification is logged with user, timestamp, and action
